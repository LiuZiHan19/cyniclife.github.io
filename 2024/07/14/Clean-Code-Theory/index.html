Clean Code Theory

Author: Liu Date: July 14, 2024  8:05:51 Category: .Net

Why Clean Code Is Essential

  • 任何傻瓜都能写出计算机能理解的代码。优秀的程序员写出的是人类能理解的代码。

    Boy Scout Rule
    让代码比发现时更好，持续改进，保持专业。

    Broken Windows Theory
    环境中的混乱和不当行为的可见迹象会鼓励进一步的混乱和不当行为，导致严重犯罪。
    似微不足道的问题或缺陷可能导致更大的问题。
    忽视的文化和对细节的关注不足。
    对代码负责。

    YAGNI Principle
    你不会需要它
    最小化偶然复杂性
    保持专注

    KISS Principle
    保持简单，傻瓜
    简洁应该是设计的主要目标
    简单=更好的用户体验

    Principle of Least Surprise
    每个组件都应该以大多数用户期望的方式行为。
    消费者应该相信他们的直觉
    明显的行为应该被实现

    Coupling
    改变一个元素需要改变其他元素。
    更改成本高昂
    解耦不是免费的

    Cohesion
    衡量元素为了实现目标而一起工作的程度。
    高凝聚力是一种可取的属性
    专注的模块

    Readability vs Performance
    编写干净的代码
    用测试覆盖它
    提高性能

    What is Duplication
    不要重复自己（DRY）
    不等同于“相同文本”
    重复的行为
    一起更改

    Idiomatic Code
    惯用法和标准
    社区一致性
    减少摩擦
    一致性

    理解
    代码是人所阅读，不是机器。
    持续进行小的改进同时评估风险。
    对每一行代码负责，保持干净，细小的问题堆积会造成大问题，这是一笔债务。
    保持一个对象拥有应有的核心属性，专注于对象本身。
    一定程度从用户角度出发设计功能。
    在解耦和耦合代码取一个最佳值。
    不同代码相同行为=重复的行为（不代表是重复的文本）应该被改进。
    总体进行一个平衡。
